# Industry DNA: 协同办公与生产力 (Collaboration Tools)
ID: ind_008_collaboration | Version: 3.0 (14D Aligned)

## [Module 1] Baseline Vector (14D)
- D8 (Social): 0.95 (协作即社交，自带传染性)
- D5 (Entry): 0.90 (极致顺滑，支持外部链接加入)
- D6 (Monetize): 0.45 (典型的 PLG 变现，依靠深度限制或团队人数收费)
- D3 (Interact): 0.85 (UI/UX 的流畅度是生产力的代名词)
- D9 (Consistency): 0.70

## [Module 2] Industry Anchors
- **D8 (Social Virality)**: 0.85=生存线。产品内部必须包含能让用户自发分享给同事的机制（如 Slack 邀请、Notion 模板）。
- **D5 (Entry Ease)**: 0.80=及格线。必须支持一键导入主流格式（如 PDF, Markdown, XLSX）。
- **D6 (Monetize Pressure)**: 0.35=标准。不能太早收费，需要等到用户形成“存档焦虑”或“协作依赖”。

## [Module 3] Critical Dimensions
- **Primary**: D8 (社交与病毒传染性), D5 (准入顺滑度)
- **Secondary**: D3 (交互), D6 (变现路径)

## [Module 4] Semantic Mappings
- "一键邀请链接" -> D8 +0.3
- "实时共同编辑" -> D3 +0.2, D8 +0.1
- "支持导入 Notion/Jira" -> D5 +0.3

## [Module 5] Physics Laws
- **The Collaboration K-Factor**: D8 超过 0.8 后，每提升 0.1，CAC 降低 40%。
- **The Empty Sandbox Trap**: IF D8 < 0.6 AND D5 > 0.8 -> TRIGGER Growth_Stagnation (如果进来容易但没有社交属性，产品会像荒漠一样迅速死寂)。

## [Module 6] Probing Logic
- IF D8.sigma > 0.5 -> ASK "当一个用户觉得好用时，产品内有什么机制能自然地迫使他邀请同事？是基于权限协作还是基于内容分享？"

## [Module 7] Macro Modifiers
- TechDebt λ: 0.7 (由于涉及多端同步，中间件一致性债务较多)
- CAC Base: 低到中 (主要靠自增长)
- Retention: 极高 (一旦团队数据沉淀，迁移成本呈指数级增长)
