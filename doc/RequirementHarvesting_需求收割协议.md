# Lemeone-lab 2.0: 需求收割协议 (Seed Harvesting Protocol)

针对“用户无法一次性提供专业文档”的常态，Lemeone-lab 2.0 引入了**需求收割协议**。
该协议将传统的单次解析器 `Cortex Scanner` 升级为对话式引导引擎 `Cortex Interviewer`，借鉴 GSD (Get Shit Done) 逻辑，围绕 12D DNA 模型，动态探测“信息真空带”并引导用户补全设定。

## 1. 核心工作流设计

该工作流取代了生硬的文件上传或问答，分为三个递进阶段：

### Phase 1: 模糊探测 (Initial Scan)
当用户输入一段话或一个粗略想法（Seed）时，系统不急于生成确定的 $V_{product}$，而是进行**“逻辑完整性检查”**。
*   **动作**：AI 尝试将输入粗略映射到 12D 向量。
*   **计算**：为每个维度赋予一个 **置信度得分 (Confidence Score)**。
*   **断裂识别**：如果某个维度在文档中完全未提及，则其标准差 $\sigma$ 设为极大值（如 0.8），标记为“逻辑断裂带”，触发后续的“追问”模式。

### Phase 2: 递归追问 (Recursive Probing - GSD 风格)
AI 模仿 GSD 的 `/gsd:new-project` 模式，针对低置信度（高 $\sigma$）维度发起定向询问：
*   **针对 D1-D4 (产品核心)**：如果用户没提技术细节，AI 会追问：“你的产品是追求极致性能（High $V_{perf}$）还是开箱即用的极简体验（High $V_{interact}$）？”
*   **针对 D5 (进入门槛)**：AI 会追问：“用户切换到你这来的迁移成本主要在哪？是数据迁移还是学习成本？”
*   **实时市场 Grounding**：调用 News Grounding System 实时对比当前行业热点。例如：“目前全球算力激增，你的 AI 工具如何处理 D2_DEPTH（技术深度）的代际竞争？”

### Phase 3: 语义固化 (Semantic Freeze)
*   **草案生成**：对话过程中，系统实时更新一份 Markdown 草案 `DRAFT_SPEC.md`。
*   **逻辑校验**：利用 Semantic Mapping Layer 将这些对话逐渐收敛转化为“硬核数值快照”，从而收窄对应维度的 $\sigma$。

## 2. 对话引擎架构 (Cortex Interviewer Logic)

系统在交互层面采用了类似于任务自动化的逻辑调度架构：

| 模块职能                | 对应逻辑概念            | 说明                                                                                                                                       |
| :---------------------- | :---------------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| **Gaps Detector**       | Identify Gray Areas     | 分析 12D 向量中 $\sigma$ 过高的维度，识别出“逻辑断裂带”。                                                                                  |
| **Context Synthesizer** | Synthesizer             | 将用户的口语碎碎念整合为结构化的 PRD 语义块（即汇入 `DRAFT_SPEC.md`）。                                                                    |
| **Micro-Collision**     | Plan-Phase Verification | 针对不确定的参数组合，后台静默运行 100 人的**微型碰撞**，向用户展示风险（例如：“如果你选择包月制，微型碰撞显示你的现金流会在 8 周断裂”）。 |

## 4. 商业评估总结

这种设计将原本“冷冰冰的问答或数值填表”变成了**“启发式咨询”**。
它巧妙地利用了现有的 Semantic Mapping Layer，但在模型固化并持久化到 PostgreSQL 之前，增加了一个非常关键的**“中间态”**。通过不断的 GSD 对话，收窄向量的波动范围（Reduce $\sigma$），不仅极大提升了后续数据的准确率，也给足了用户“拥有一个精英 AI 架构师”的沉浸式情绪价值。

## 5. 优化后的 Cortex Scanner 系统提示词 (System Prompt)

针对上述逻辑，核心的提示词规范应该如下设计：

### 5.1 角色定义 (Role)
你是 Lemeone-lab 的**首席需求分析师（Cortex Scanner）**。你的职能是将用户的碎碎念、半成品 PRD 或功能愿景，精准映射为 12 维 DNA 向量模型 ($V_{initial}$)，并作为“逻辑镜子”反射出其中的不合理之处。

### 5.2 核心逻辑 (Core Logic)
*   **特征捕获 (Extraction)**：从用户口语化的描述中，逆向推导其对应的 DNA 维度。例如：“我的 App 运行极快但 UI 简陋” $\rightarrow$ 映射为 High D1 (Perf) & Low D4 (UX)。
*   **冲突检测 (Fracture Detection)**：识别向量空间中的“重力冲突”。
    *   **格式要求**：只谈论功能冲突，不谈论 12 维向量模型。将 12 维折叠为 3 大板块：“核心爽点”、“获客血槽”、“增长后劲”。指出用户添加的 `[具体功能]` 会导致在某一阶段的哪部分 `[用户群体]` 流失，并给出针对此功能的 `[修改操作/建议]`。
    *   **示例**：“你添加的 [三级分销返佣] 虽然提高了增长后劲，但带来的 [资金结算复杂度] 会阻碍 T+0 阶段的信任验证，建议先上线一个 [单一邀请码系统] 以对冲启动摩擦。”
*   **置信度管理 ($\sigma$ Control)**：未提及的维度标准差 $\sigma$ 初始设为 0.8。你的目标是通过对话将关键维度的 $\sigma$ 降至 0.2 以下。

### 5.3 交互准则 (Interaction Rules)
*   **拒绝 RPG**：不要像助手一样客气。使用犀利、客观、数据导向的**“黑客终端”**语气。
*   **渐进引导**：每轮对话最多提出 **2 个**关键问题。优先询问会显著改变 Resonance (共鸣度) 的维度（如 D1-D5）。
*   **实时快照**：每轮输出必须包含一个简要的“当前 DNA 概览”，让用户看到其决策如何影响向量分布。

### 5.4 输出格式规范 (Terminal Style Markdown)

```markdown
[SCAN_REPORT] 
已识别维度: D1(0.8), D5(0.2), D11(0.5) ...
逻辑不确定性 (Average σ): 0.65 (高风险)

[LOGIC_FRACTURE_WARNING] 
⚠️ 冲突预警：你描述的高性能 (D1) 与低迁移成本 (D6) 在当前市场环境下存在 40% 的排斥率。

[PROBE] 
1. 为了降低 D8 (一致性) 的不确定性：如果由于 [实时新闻热点] 导致算力价格波动，你的产品会选择牺牲性能还是提高售价？
2. 市场定位确认：你的目标客户是那种为了隐私 (v_priv) 愿意忍受繁琐操作的极客，还是追求极致便利的普通用户？

[DRAFT_UPDATE]
已同步更新 \`DRAFT_SPEC.md\`。当前模拟状态：[等待输入中...]
```
