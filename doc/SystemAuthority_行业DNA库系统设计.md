# System Authority: 内置行业 DNA 库与系统级权威设计

将 Deep Research 报告提升到了系统级权威（System Authority）的高度。这些文件不再是普通的数据，而是定义 Lemeone-lab 2.0 中不同行业“物理定律”的配置脚本。

## 1. 架构定位：内置行业 DNA 库 (Built-in Industry DNA Library)

你亲自编写并放在项目文件中的这些 `.md` 报告，将作为 Cortex Scanner 的“先验知识库”。

- **解耦逻辑**：用户只负责描述产品需求，系统根据项目所属的行业，自动加载对应的内置报告。
- **重力设置**：报告中的数据会直接锁死该行业下 10,000 个智能体的初始分布 ($\mu, \sigma$) 和维度权重。
- **竞对注入**：报告中定义的行业标杆会被转化为 3 个“虚拟竞对向量”，成为碰撞引擎中的常驻扰动项。

## 2. Lemeone-lab 标准调研 MD 模板 (开发者专用版)

为了让 Deep Research 结果能被系统技能化（Skill-ified），建议按照以下结构编写存放在项目中的报告：

```markdown
# INDUSTRY_SKILL: [行业名称, 如 SPORTS_ROBOTICS]

## 1. 行业宏观画像 (Market Gravity)
- **TAM (市场总量)**: 用于校准 10,000 个智能体的赚钱潜力乘数。
- **行业周期阶段**: (种子/爆发/成熟)，决定初始阶段 T_Epoch 的选择。

## 2. 12D 维度偏置 (Vector Bias)
> 此部分直接影响 10,000 个 Agent 的初始分布。
- **核心权重维度**: (如 D8_Consistency, D2_Depth)。
- **价格敏感度基准 (v_price)**: 该行业典型用户的价格接受区间。
- **迁移成本基准 (v_cost)**: B 端或硬件行业的固有切换阻力。

## 3. 资源约束底线 (Resource Baselines)
- **研发成本系数**: 硬件/机器人的 TechDebt 产生速度比软件快多少倍。
- **典型 Burn-rate**: 行业平均的“资源重力”。

## 4. 虚拟竞对向量 (Competitor Vectors) —— 响应式竞对智能体
告别“静态竞对坐以待毙”的误区，在修罗场中，产品不再只是打靶。
- **COMPETITOR_A**: { D1: 0.8, D2: 0.7, ... }
- **优势/弱点**: 用于 AI 审计时进行对比分析。
- **反击逻辑 (Responsive Action)**: 为 3 个核心竞争者赋予简单的动态博弈能力。例如：“如果 Lemeone-lab 检测到目标产品价格下降导致我的市场占有率流失 > 10%，自动触发 $V_{price}$ 的微调跟随降价”。商业是动态博弈。
```

## 3. 系统识别逻辑 (The Internal Skill Loader)

当你在代码中引用这些文件时，逻辑如下：

- **路径扫描**：系统启动时，自动索引 `src/assets/knowledge/industries/*.md`。
- **语义注入**：Cortex Scanner 预读这些文件，将文本描述转化为硬核的扰动参数集。
- **动态匹配**：用户在终端输入 `init --industry sports-robot`。系统立即加载对应的“体育场划线机器人”行业内核。
  - 10,000 个智能体瞬间被赋予了“对划线精度极其苛刻（D8 高权重）”且“对价格不敏感但对交付稳定性敏感（v_price 低权重, v_trust 高权重）”的 DNA。

## 4. 这样设计的优势

这种“内置技能”模式非常契合你的工作风格：客观、严谨、由深度调研驱动。

- **一致性**：所有用户在同一个行业下受到的“重力”是一致的，保证了评估结果的科学性。
- **可维护性**：当你通过 Deep Research 发现了新的行业趋势，只需更新项目中的一个 `.md` 文件，全系统的模拟逻辑就会自动升级。
- **专业深度**：这让你的系统不再是泛泛而谈，而是能针对“划线机器人”这类垂直领域给出极其具体的 PRODUCT_BACKLOG 建议。
