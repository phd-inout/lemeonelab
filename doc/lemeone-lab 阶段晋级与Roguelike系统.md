# 公司阶段晋级系统设计文档

> **核心哲学**：晋级不只是数值门槛的跨越，而是**"主要矛盾的转移"**。
> 每个阶段，创始人面临的根本问题完全不同——这才是最真实的创业模拟。

---

## 一、准入闸门系统（The Gate System）

每次阶段晋级必须同时满足三个维度，缺一不可：

```typescript
interface StageGate {
  hardStats: HardStatRequirement[]    // 硬性数值门槛
  vectorThreshold: AttributeRequirement  // 创始人属性要求
  keyIncident: KeyIncidentType        // 必须经历的核心事件
}
```

**设计意图**：防止"刷进度条"式的无脑推进。仅靠堆时间不够，必须有能力、有故事。

## 二、各阶段详细定义

### Stage 1 → SEED → MVP

**主要矛盾**：「我有想法，但没有东西能拿出去展示」

**目标**：从点子到一个可以被别人用的东西

| 维度 | 要求 | 说明 |
|------|------|------|
| 硬性-产品完成度 | `devProgress ≥ 25%` | 核心功能跑通 |
| 硬性-护城河 | `moat ≥ 10` | 初步技术差异化 |
| 属性阈值 | `TEC > 60` 或 `LRN > 70` | 有能力把想法落地 |
| 核心事件 | `[FIRST_CONTACT]` | 获得 ≥ 10 个意向种子用户 |

**特殊机制：小确幸系统**
SEED 阶段是最难熬的时期，概率为正向事件增加 30% 权重（代码层用 `positivityBoost: 1.3`），防止玩家在引擎刚冷启动时就被负向事件劝退。

```typescript
// SEED 阶段事件加权
function pickEventsForSprint(state, weeks) {
  if (state.company.stage === 'SEED') {
    return pickWithPositivityBoost(state, weeks, 1.3)
  }
  // ...
}
```

---

### Stage 2 → MVP → PMF

**主要矛盾**：「产品做出来了，但没人愿意真的掏钱」

**目标**：验证"有人愿意买单"

| 维度 | 要求 | 说明 |
|------|------|------|
| 硬性-产品完成度 | `devProgress ≥ 60%` | 功能相对完善 |
| 硬性-首笔收入 | `totalRevenue ≥ ¥1` | 哪怕一块钱，代表市场验证 |
| 硬性-现金跑道 | `cashRunway ≥ 12 周` | 不含融资的自生存能力 |
| 属性阈值 | `MKT > 50` 且 `CHA > 60` | 具备说服用户的能力 |
| 核心事件 | `[THE_CLICK]` | 周留存率模拟 ≥ 30%（由 AI 事件触发判定） |

**The Click 事件设计**：
```json
{
  "id": "key_incident_the_click_001",
  "name": "The Click — 用户真的留下来了",
  "stage": ["MVP"],
  "category": "MARKET",
  "isKeyIncident": true,
  "keyIncidentId": "THE_CLICK",
  "baseProbability": 0,
  "triggerCondition": "devProgress >= 60 && mrr > 0 && mkt > 50",
  "effects": [
    { "target": "moat", "delta": 15 },
    { "target": "mrr", "delta": 3000 }
  ],
  "narrativePrompt": "描述创始人第一次看到真实用户自发回来使用产品时的震惊与喜悦，要有画面感和细节"
}
```

---

### Stage 3 → PMF → SCALE

**主要矛盾**：「有了持续收入，但靠我一个人扩不出去」

**目标**：从"生意"变成"系统"——把个人能力转化为可复制的流程

| 维度 | 要求 | 说明 |
|------|------|------|
| 硬性-收入 | `mrr ≥ 盈亏平衡点` | 覆盖所有支出，正现金流 |
| 硬性-护城河 | `moat ≥ 40` | 具备抗抄袭能力 |
| 属性阈值 | `FIN > 60` 且 `OPS > 50` | 能管理复杂现金流和流程 |
| 核心事件 | `[THE_INVESTMENT]` | 完成首轮正式融资（天使/种子轮） |

**人才短缺危机**：进入 PMF 晋级判定期时，系统强制触发一次「人才短缺」事件（不可规避），逼迫用户做是否雇佣的决策。这是引入 HR 系统的最自然时机。

---

### Stage 4 → SCALE → IPO

**主要矛盾**：「规模上去了，但系统和合规跟不上」

**目标**：标准化与合规化，从"草台班子"到"可审计的公司"

| 维度 | 要求 | 说明 |
|------|------|------|
| 硬性-年收入 | `annualRevenue ≥ ¥70,000,000` | 约合 $10M |
| 硬性-声誉 | `reputation ≥ 70` | 行业知名度（新增指标，见下方） |
| 属性阈值 | `OPS > 80` | 系统性规避风险的能力 |
| 核心事件 | `[COMPLIANCE_PASS]` | 通过严苛的外部财务与合规审计 |

**对话设计（防无聊机制）**：SCALE 阶段引入「大厂博弈」和「公关危机」高频事件（每 3 sprint 必触发一次），让玩家无法进入自动驾驶状态。

---

### Stage 5 → IPO → TITAN

**主要矛盾**：「上市了，但随时可能成为被猎杀的目标」

**目标**：定义行业标准，成为底层基础设施

| 维度 | 要求 | 说明 |
|------|------|------|
| 硬性-市场占比 | `marketShare ≥ 35%` | 核心细分领域第一 |
| 硬性-生态规模 | `ecosystemConnections ≥ 1000` | 第三方接入数（AI 时代特有指标） |
| 属性阈值 | `CHA > 90` | 全球影响力，不只是产品力 |
| 核心事件 | `[THE_SINGULARITY]` | 技术成为行业底层基础设施 |

### 3.2 里程碑奖励（Perk Tree）

达到每个阶段 **50% 进度**时触发一次奖励选择（不是 Stage 晋级，是中途奖励）：

```typescript
type MilestonePerk =
  | 'ATTRIBUTE_REROLL'    // 随机重新分配 30 个属性点
  | 'BW_UPGRADE'          // 带宽上限永久 +10
  | 'CASH_INJECT'         // 一次性注入相当于 3 个月 burn rate 的资金（模拟天使追投）
  | 'INSIGHT_FOCUS'       // 下一次 Aha-Moment 必定触发（保底机制）
  | 'MOAT_BOOST'          // 护城河永久 +20（产品壁垒奖励）
```

**终端交互**：
```
🎯 里程碑达成！你距离 PMF 已完成 50% 的任务。
   请选择你的奖励（只能选 1 个）：

   [1] 属性洗点  — 重新分配 30 个属性点
   [2] 带宽提升  — 带宽上限永久 +10

   [5] 护城河加固 — Moat 永久 +20

→ 选择 [1-5]:
```

## 四、Roguelike Meta Progression 系统（遗产系统）

> **核心哲学**：在 lemeone-lab 里，失败不是终点，而是下一次更强大的起点。
> 每一局模拟都是"一代创始人"。你的遗产会传递给"下一代继承者"。

### 4.1 遗产积分（Legacy Points）

```typescript
interface LegacyRecord {
  founderName: string
  finalStage: CompanyStage     // 死在哪个阶段
  highestValuation: number
  weeksAlive: number
  legacyPoints: number         // 本局获得的遗产积分
  ahaInsightsUnlocked: string[]  // 本局触发过的 Aha-Moment ID
}
```

**遗产积分计算公式**：
```
LegacyPoints = (stageScore × 100) + (weeksAlive × 2) + (higherValuation / 10000)
  + ahaInsightsBonus（每触发 1 个 Aha-Moment +50）
  + optionalChallenge（接受了某些高难度事件加成）

stageScore: SEED=1, MVP=3, PMF=6, SCALE=12, IPO=20, TITAN=35
```

### 4.2 遗产积分兑换（Meta Upgrades）

遗产积分在重开新局时可以花费，分为三个层级：

#### 🟢 永久属性加成（跨局）
```
花费 50  → 新局创始人 任意 1 维属性初始值 +5
花费 100 → 新局创始人 带宽上限 +5
花费 150 → 新局创始人 初始现金 +¥10,000
花费 200 → 新局创始人 自选 1 维属性 +10
```

#### 🔵 开局特权（一次性）
```
花费 80  → 新局跳过 SEED→MVP 核心事件检验（默认 [FIRST_CONTACT] 已完成）
花费 120 → 新局开局解锁"遗产人脉"（初始 CHA 视为 +20 效果，不改变属性值）
花费 200 → 新局开局有 1 位免费初始员工（DEV，talent=60）
```

#### 🔴 传奇遗产（高价，强力）
```
花费 500 → 解锁"硅谷老炮"特殊背景
           极高 CHA(80) + OPS(75)，初始资金 ¥500,000
           代价：TEC 永久上限降至 70（老一代对新技术不敏感）

花费 800 → 解锁"遗产公司"模式
           继承上一局公司的 moat 值（最高继承 50%）
           相当于你的新公司"站在前辈的肩膀上"

花费 1000 → 解锁"Aha Archive"
            上一局触发过的所有 Aha-Moment 在新局中会出现在 archive 里
            可随时查阅，相当于一个私人商业智慧库
```

### 4.3 "传记模式"（Bio Mode）

对于第 5 局以上的老玩家，系统解锁「传记模式」：

```
新局启动时，AI 会为当前创始人生成一段"前传"：

"你叫 Derek，上一代创始人 Marcus 的学生。
 Marcus 在 2029 年的 SCALE 阶段因为管理内耗崩溃，留给了你：
   - 一套尚未完善的 AI 协议代码（moat +15 加成）
   - ¥80,000 遗产现金
   - 一份来自 Marcus 的手写信：'别像我一样忽视 OPS。'

 你的历史正在继续……"
```

这让每一局都有叙事连续性，极大增强情感连接和留存动力。

### 4.4 Roguelike 核心设计原则

| 原则 | 实现方式 |
|------|---------|
| **死亡是必然的，但不是惩罚** | 遗产积分确保失败有收益 |
| **每局都不一样** | 背景×行业×商业模式组合 = 180 种开局 |
| **每次都能变得更强** | Meta 升级不设上限，但边际收益递减 |
| **新人能玩，老手有深度** | 传记模式、传奇遗产只对多周目玩家开放 |
| **失败要有意义** | AI 尸检报告 + Aha Archive = 每次失败都是知识 |

## 五、Reputation（声誉）新增指标设计

SCALE 阶段引入了 `reputation`，这是一个新的公司指标，补充说明：

```typescript
// 声誉值（0-100）
// 影响因素：
//   + 每次成功的 PR 事件 / 产品奖项 (+5 to +15)
//   + 行业媒体正面报道 (+3 to +8)
//   - 公关危机事件 (-15 to -30)
//   - 创始人 Burnout 导致的公开失态 (-10)
//   - 员工差评平台曝光 (-8)

// 影响效果：
//   reputation > 70: 融资条件改善（投资人主动联系）
//   reputation > 85: 顶级人才主动投递简历
//   reputation < 30: 触发"信任危机"事件链
```

## 六、商业化埋点

### 6.1 加速券（算力配额消费）

| 场景 | 效果 | 建议定价 |
|------|------|---------|
| IPO 合规审计加速 | 压缩 4 周等待期至 1 周 | 50 算力积分 |
| 融资流程加速 | 跳过 2 周谈判期 | 30 算力积分 |
| 顿悟保底 | 下一次 Aha-Moment 必定触发 | 20 算力积分 |

### 6.2 专家合伙人（临时购买）

```
$ hire --expert venture-advisor

临时雇佣"风险投资专家 AI 顾问"（由 Gemini Pro 驱动）
有效期：1 次关键晋级决策
费用：100 算力积分

顾问将分析你当前的所有指标，给出 100% 针对当前状态的
最优策略建议（不会像普通 Aha-Moment 一样"刻意模糊"）。

⚠️ 注意：过度依赖顾问会减少你的"遗产积分"加成（学习值降低）
```

### 6.3 声誉保护（Graveyard 回避）

- **逻辑**：破产前消费 200 算力积分购买"公关紧急干预"
- **效果**：`isFailed` 仍为 true，但 `LeaderboardEntry.failedReason` 被清空，不进 Graveyard 墓碑区
- **设计意图**：面子经济，真实创业者最在乎的就是"公开记录"
